🏋️ Agent-oriented · MCP · Telegram Mini App

Tu entrenador vive en Telegram.

No es una app de gimnasio más. Es un coach de IA que te conoce, crea tus entrenos y te guía serie a serie — todo por chat.

⬇ Ver en GitHub 🚀 Cómo montarlo

¿Cómo funciona?

Hablas con tu coach en Telegram. Él te pregunta, crea el plan y te lo abre. Tú entrenas y registras series en la Mini App. El coach ajusta en tiempo real.

Le dices que vas a entrenar

«Voy a entrenar». El coach pregunta energía, tiempo y molestias.

Él crea el plan adaptado

Basado en tu perfil, historial y energía del día. 4-6 ejercicios, series y pesos sugeridos.

Te abre la Mini App

Un botón en el chat abre el plan visual. Ves cada ejercicio, gif, series. Registras tocando.

Él ajusta en vivo

¿Máquina ocupada? ¿Molestia? Le dices por chat y cambia el ejercicio al instante.

Al terminar, feedback

Le cuentas cómo fue. Guarda todo. La próxima vez adapta mejor.

¿Por qué es diferente?

🤖 Agente, no plantilla

Cada entreno es único. El coach razona sobre tu perfil, no escupe una rutina pre-hecha.

💬 Todo por chat

No rellenas formularios. Hablas. El coach hace todo: plan, series, cambios, feedback.

🔌 MCP = control total

El agente controla la app entera via 18 herramientas MCP. La app es agnóstica al agente.

🔒 Tus datos son tuyos

Auth via Telegram. Cada usuario ve solo lo suyo. Despliegas tu propia instancia.

Arquitectura

Tú ──Telegram──> Coach (Hermes) ──MCP──> API (FastAPI) ──> Postgres │ └──deep link──> Mini App (visual)

La app no sabe quién es el coach. Solo expone herramientas MCP. Cualquier agente con soporte MCP puede controlarla.

Montar tu propio coach

Dos opciones: Docker Compose local (5 min) o Coolify con dominio.

Opción A — Docker Compose

git clone https://github.com/jlfernandezfernandez/gym-tracker.git cd gym-tracker cp .env.example .env # pon tu TELEGRAM_BOT_TOKEN docker compose up -d # app en localhost:8000

Opción B — Coolify

# 1. Fork del repo en GitHub # 2. Coolify → New Resource → GitHub → gym-tracker # 3. Añadir Postgres como servicio vinculado # 4. Env vars: TELEGRAM_BOT_TOKEN, CORS_ORIGINS # 5. Deploy

Conectar el coach (Hermes)

hermes profile create gym-coach hermes -p gym-coach mcp add gym_tracker \ --stdio mcp/gym_tracker_mcp.py \ --env GYM_TRACKER_API_BASE=https://gym.tu-dominio.com/api hermes -p gym-coach config set telegram.bot_token "TOKEN" hermes -p gym-coach gateway start

Lee docs/coach-setup.md para la guía completa.
